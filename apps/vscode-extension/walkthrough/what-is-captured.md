Recall watches a handful of signals as you work:

- **File saves** — captured as a diff against the previous saved version, never the full file content.
- **Terminal commands** — the command line, working directory, exit code, and a truncated excerpt of the output.
- **Debug sessions** — the launch config name and any exceptions hit while debugging.
- **Git activity** — commits and branch switches, via VS Code's built-in Git extension.
- **Resolved errors/warnings** — the moment a diagnostic in the Problems panel clears.
- **Task runs** — build/test/watch tasks and their exit codes.
- **Manual notes** — anything you explicitly save with `Recall: Save as Memory`.

Nothing here ever leaves your machine.
